#### 3. Reference.php - PERFECT (No Changes Needed)

**File:** `app/Models/Reference.php`

**Status:** ✅ Sudah perfect dari awal, termasuk helper method

**Isi Model:**
```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use App\Enums\OperatorType;

class Reference extends Model
{
    protected $fillable = [
        'name',
        'material_id'
    ];

    public function material()
    {
        return $this->belongsTo(Material::class);
    }

    public function specifications()
    {
        return $this->belongsToMany(
            Specification::class,
            'reference_specification',
            'reference_id',
            'specification_id'
        )->withPivot([
            'operator',
            'value',
            'max_value',
            'text_value'
        ])->withTimestamps();
    }

    public function samples()
    {
        return $this->hasMany(Sample::class);
    }

    // Helper get specification value
    public function getSpecValue($specificationId)
    {
        $spec = $this->specifications()->where('specification_id', $specificationId)->first();

        if (!$spec) {
            return null;
        }

        return [
            'operator' => $spec->pivot->operator,
            'value' => $spec->pivot->value,
            'max_value' => $spec->pivot->max_value,
            'text_value' => $spec->pivot->text_value,
        ];
    }
}
```

**Fitur:**
- ✅ Fillable: name, material_id
- ✅ Relations: belongsTo material, belongsToMany specifications (pivot), hasMany samples
- ✅ Helper: getSpecValue($specificationId) untuk ambil spec value dari pivot

## 3. HANDOVER FLOW (RECAP)

### Flow Lengkap:

**1. Sample Status = "In Progress"**
- Primary analyst sedang mengerjakan sample
- Button "Submit to Hand Over" tersedia

**2. Submit to Hand Over:**
- User klik "Submit to Hand Over"
- Pilih operator tujuan (to_analyst_id)
- Isi reason (required) dan notes (optional)
- **Sample status → "Hand Over"**
- Create record di sample_handovers (status = 'pending')
- Sample hilang dari dashboard user yang submit

**3. Take Over:**
- Operator tujuan login
- Lihat notifikasi "Sample Handover Menunggu Anda"
- Klik "Ambil Sample"
- **Sample status → "In Progress" (kembali)**
- Handover status → 'completed'
- Sample primary_analyst_id → operator baru
- Sample muncul di dashboard operator baru

### Status Transition:
```
Pending → In Progress → Hand Over → In Progress → Completed
            ↑              ↓           ↑
       (Start)        (Submit)    (Take Over)
```

### Key Design Decisions:

**1. Universal Tables:**
- Single `samples` table untuk raw_material & solder (bukan 2 terpisah)
- Single `categories` table untuk raw_material & solder
- Single `materials` table untuk raw_material & solder
- Single `references` table untuk raw_material & solder
- Benefit: No duplication, easier maintenance, support analytics

**2. String vs Enum for sample_type:**
- Using STRING not ENUM
- Benefit: Flexibility, no ALTER TABLE to add new types
- Validation di Livewire level

**3. Normalized Handovers:**
- Separate `sample_handovers` table
- Benefit: Multiple handovers per sample, historical tracking
- Clean samples table (30 columns instead of 40)

**4. Status Integration:**
- New status "Hand Over" (name: hand_over)
- Sample status changes during handover flow
- Clear visibility: operator lama tidak lihat, operator baru belum lihat

**5. Direct Assignment Handover:**
- to_analyst_id REQUIRED (not nullable)
- Not open handover, not first-come-first-served
- Pasti ada operator tujuan
- No shift tracking (too detailed)

---

**1. Accessing Relations:**
